# 貢献ガイドライン

IBM Bob Linksプロジェクトへの貢献に興味を持っていただき、ありがとうございます！

## ⚠️ 重要な注意事項

このプロジェクトは**コミュニティ主導**で運営されており、以下の点にご注意ください：

- **サポートはベストエフォート**: メンテナーは限られた時間で対応しています
- **即座の対応は保証されません**: IssueやPRへの返信に時間がかかる場合があります
- **すべての提案が採用されるわけではありません**: プロジェクトの方向性に合わない場合があります
- **自己責任**: コードの変更や使用は自己責任で行ってください

### 2. プルリクエスト

#### 事前準備

1. **フォークとクローン**
   ```bash
   # リポジトリをフォーク後
   git clone https://github.com/YOUR_USERNAME/IBMBob.git
   cd IBMBob
   ```

2. **ブランチを作成**
   ```bash
   git checkout -b feature/your-feature-name
   # または
   git checkout -b fix/your-bug-fix
   ```

#### コーディング規約

- **JavaScript**: 
  - ES6+ の構文を使用
  - セミコロンを使用
  - 2スペースインデント
  - 意味のある変数名・関数名

- **HTML/CSS**:
  - セマンティックHTML
  - BEM命名規則（推奨）
  - レスポンシブデザイン

- **コメント**:
  - 複雑なロジックには日本語でコメント
  - JSDocスタイルの関数コメント

#### コミットメッセージ

以下の形式を推奨：

```
<type>: <subject>

<body>
```

**Type**:
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメントのみの変更
- `style`: コードの意味に影響しない変更（空白、フォーマットなど）
- `refactor`: バグ修正や機能追加ではないコード変更
- `test`: テストの追加や修正
- `chore`: ビルドプロセスやツールの変更

**例**:
```
feat: add article filtering by tag

Qiita記事をタグでフィルタリングする機能を追加。
ユーザーは特定のタグの記事のみを表示できるようになります。
```

#### プルリクエストの作成

1. **変更をプッシュ**
   ```bash
   git add .
   git commit -m "feat: add new feature"
   git push origin feature/your-feature-name
   ```

2. **PRを作成**
   - GitHubでPRを作成
   - 以下のテンプレートを使用：

```markdown
## 変更内容
[変更の簡潔な説明]

## 関連Issue
Closes #[Issue番号]

## 変更の種類
- [ ] バグ修正
- [ ] 新機能
- [ ] ドキュメント更新
- [ ] リファクタリング
- [ ] その他: [説明]

## テスト
- [ ] ローカルでテスト済み
- [ ] 複数のブラウザで確認
- [ ] モバイルで確認

## チェックリスト
- [ ] コードが規約に従っている
- [ ] コメントを追加した（必要に応じて）
- [ ] ドキュメントを更新した（必要に応じて）
- [ ] 既存の機能を壊していない

## スクリーンショット
[可能であれば添付]
```

### 3. レビュープロセス

- **返信に時間がかかる場合があります**: メンテナーは限られた時間で対応しています
- **変更を求められる場合があります**: コードレビューで改善点が指摘されることがあります
- **マージされない場合があります**: プロジェクトの方向性に合わない場合があります

## 📋 開発環境のセットアップ

### 必要なツール

- Git
- テキストエディタ（VS Code推奨）
- Webブラウザ（Chrome、Firefox、Safari）
- ローカルHTTPサーバー（オプション）

### ローカルでの実行

```bash
# リポジトリをクローン
git clone https://github.com/ayatokura/ibmbob-links.git
cd IBMBob

# ローカルサーバーを起動（オプション）
python3 -m http.server 8000
# または
npx http-server -p 8000

# ブラウザで開く
open http://localhost:8000
```

### テスト

現在、自動テストはありません。以下を手動で確認してください：

- [ ] すべてのページが正しく表示される
- [ ] Qiita APIから記事が取得できる
- [ ] キャッシュ機能が動作する
- [ ] エラーハンドリングが適切
- [ ] レスポンシブデザインが機能する
- [ ] 複数のブラウザで動作する
